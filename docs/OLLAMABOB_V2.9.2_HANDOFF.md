## 3. What this session changed (V2.9.2 — uncommitted)

### Problem reported by user
"I tried to have Bob check my Mail. He said `Error: Not authorized to send Apple events to Mail. (-1743)`. I clicked Approve on the macOS prompt — he still says no can. Also, can we enable this type of stuff during onboarding?"

### Root cause
`Info.plist` generated by `build.sh` was missing **`NSAppleEventsUsageDescription`**. Without that TCC string, macOS accepts the user's "Approve" click but the subsequent Apple-event send still fails with `-1743`. The folder-access usage strings were present; the Apple-events one was not.

### Fix + feature (in the working tree, ready to commit)

| File | Change |
|------|--------|
| `OllamaBob/build.sh` | Added `NSAppleEventsUsageDescription`, `NSContactsUsageDescription`, `NSCalendarsUsageDescription`, `NSRemindersUsageDescription` to the inlined Info.plist heredoc. |
| `OllamaBob/OllamaBob/Services/AutomationProbe.swift` (new, ~100 lines) | `@MainActor ObservableObject` singleton that probes Mail, Calendar, Reminders, Contacts, Music, Finder, System Events with a trivial `tell application "X" to return name` AppleScript. Classifies each as `granted` / `denied` / `missing` / `error` / `unknown`. Exposes `probeAll()`, `probe(target)`, and `openSystemSettings()`. |
| `OllamaBob/OllamaBob/Views/OnboardingView.swift` | New Permissions step (step index 2 of 4). "Grant all" button walks the targets; each row shows current status with a "Grant / Retry / Recheck / Skip" button. Step count bumped from 3 to 4 (`stepCount` constant). Window height increased `520 → 560`. |
| `OllamaBob/OllamaBob/Views/PreferencesView.swift` | New "Mac App Permissions" section in the Tools tab, sandwiched between built-in tools and the external CLI tools divider. "Re-run prompts" + "Open System Settings" buttons, plus one row per target with a "check" action. |

## 4. Feature inventory (what's shipped)

### V1 (core)
- Native SwiftUI/AppKit menu bar app targeting macOS 14.
- Direct HTTP to Ollama at `localhost:11434` via `/api/chat` (never the OpenAI-compatible endpoint).
- Agent loop in Swift (`Agent/AgentLoop.swift`), non-streaming, hard caps: 10 tool iterations, 120s total, 30s per tool.
- Original four tools: `shell`, `read_file`, `search_files`, `web_search`.
- Modal NSAlert approval for all writes; forbidden list for destructive commands; path policy for sensitive directories.
- GRDB SQLite persistence (`Persistence/Database.swift`, `Schema.swift`).
- Preflight check on launch with user-facing error view if Ollama is down.
- Primary model `gemma4:e4b`; fallback `qwen3:14b` after 3 consecutive parse failures per session.

### V2.0 → V2.4 (persona / memory / tools / onboarding)
- Five built-in personas (Mumbai Bob, Terse Engineer, Grumpy Linus, Helpful Assistant, Blank) via `Personality/PersonaStore.swift`.
- Persistent user profile + sticky facts memory (30-day LRU, identity-exempt). Write path uses a compaction extractor that runs with `keep_alive: 0`.
- Prompt composer builds: `[OperatingRules] + [Persona] + [UserProfile] + [ToolCheatSheet]`. Tier-1 tools always visible, tier-2 conditional on a 800-token budget.
- `<untrusted>` wrapper + delimiter neutering around tool output as injection defense.
- Spillout to disk for oversized tool outputs; agent reads via character offsets.
- Onboarding wizard with step dots (welcome, persona pick, tour) — now expanded to include the permissions step from this session.
- Conversation compaction at 75% context fill.
- Tool catalog in `Resources/ToolCatalog.json`, live-probed at launch by `ToolRuntime`.

### V2.5 — Make Bob Sing
- 50 pre-rendered ElevenLabs voice clips bundled under `Resources/BobSayings/` (greetings, celebrations, boasts). Offline playback — no runtime TTS.
- Regenerate via `tools/render-bob-sayings.py` (needs `ELEVENLABS_API_KEY` + `OLLAMABOB_VOICE_ID` in `.env`).
- Heartbeat ticks fire greetings/celebrations when Bob does real work; shortcuts tab; conversation tools (new/clear, pin).

### V2.6 — Clipboard + AppleScript
- `ClipboardTool.swift` (read/write) and `AppleScriptTool.swift` (modal-gated, forbidden-pattern list: no `do shell script`, no keystrokes, no mount, no admin).

### V2.7 — Swappable avatar packs
- `Models/AvatarPack.swift` and `AvatarStore`; classic robot (phosphor-green monochrome) and Mumbai cartoon (full-color). Pack selection persists per persona.
- Sprites live in `Resources/Avatars/{pack}_{mood}.png`. Use `rembg[cpu,cli]` to regenerate cutouts — generative image models cannot produce true alpha.

### V2.8 — Chromeless Bob's Desk
- Single window hosting portrait + chat. Chromeless NSWindow with `WindowDragHandle` (line ~243 of BobsDeskView.swift), `.resizable` styleMask, `hasShadow=true`, `.windowResizability(.contentMinSize)` with min + ideal frame.
- "Invisible thoughts" phosphor-green tool trace above Bob during processing; white speech bubble with his final answer when idle.
- Thinking/answer split: tool traces never pollute the chat transcript.
- Process memory readout in the status line (Ollama RAM + Bob RAM merged). Uses `ps -axo rss=,command=` — `comm=` truncates to 16 chars on macOS and never matches "ollama".

### V2.9 Phase A — Native tool expansion (shipped in `f35b72c`)

## 8. Non-negotiable architecture rules (repeat from CLAUDE.md)

If you break one of these, the user will revert the change. Do not argue with the rule — if you want to break it, open a discussion first.

1. **Endpoint is `/api/chat`** (Ollama native), never `/v1/chat/completions`.
2. **`stream: false`** everywhere. Gemma 4 + streaming + tool calls is broken.
3. **Flat tool schemas.** Single-level properties. No nested objects or arrays anywhere in the tool spec.
4. **No auto-approve.** Modal NSAlert for every write. Path policy for sensitive dirs.
5. **No Python subprocess, no MCP, no Electron, no LangChain, no Hermes Agent.** The Swift agent loop owns everything.
6. **`Codable` for tool-call arguments must accept both object and string** — Ollama flips formats between turns (see `JSONValue.swift` and `docs/OLLAMABOB_V1.1_PLAN.md`).
7. **App Sandbox OFF** — required for `Process()` shell execution. Hardened runtime ON with the usage-description + automation-events exceptions declared in `build.sh`'s Info.plist heredoc.
8. **`LSUIElement` true** — menu-bar only, no Dock icon.
9. **Persona ≠ operating rules.** Safety rules live in `BobOperatingRules` and must always be re-prepended. Voice/tone is editable; safety is not.

## 9. Known wire-format + runtime gotchas

- **Multi-turn `arguments` bug** — second tool call in a conversation may return `arguments` as a JSON string even though the first was an object. `JSONValue` must handle both.
- **Gemma 4 special chars** — backticks, braces, regex inside tool argument strings can crash the parser. Keep tool schemas simple.
- **Ollama process memory** — use `ps -axo rss=,command=`; `comm=` truncates at 16 chars.
- **AppleScript -1743 after user approves** — caused by missing `NSAppleEventsUsageDescription`. Fixed this session. If it resurfaces on rebuild, suspect ad-hoc-signing churn (§7B).
- **rembg for alpha cutouts** — `uv tool run --from 'rembg[cpu,cli]' rembg i IN OUT`. Do NOT try to use a generative image model — it paints a background instead of producing true alpha.
- **Chromeless SwiftUI windows need three pieces together** — `WindowDragHandle` NSView for dragging, `.resizable` styleMask + `hasShadow=true`, `.windowResizability(.contentMinSize)` with min + ideal frame. Remove any one and the window breaks.
