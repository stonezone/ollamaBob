### 3A. AppleScript/TCC fix + Permissions UI (in WIP `9310b0b`)

**Problem reported by user:** "I tried to have Bob check my Mail. He said `Error: Not authorized to send Apple events to Mail. (-1743)`. I clicked Approve on the macOS prompt — he still says no can. Also, can we enable this type of stuff during onboarding?"

### 3C. Chromeless redesign + dynamic bubble sizing (uncommitted)

After 3B landed as WIP, user flagged that avatar-only mode still had (i) a persistent gray `NSThemeFrame` backdrop, (ii) a speech bubble that grew over Bob on long responses, and (iii) an input pill that was a plain `Capsule` with no tail. The user supplied a target mockup (`images/how_avatar_chat_only_should_look.png`) showing: translucent bubble floating above Bob with a downward tail, Bob centered, pill input at the bottom with an **upward** tail — all over the user's desktop wallpaper with zero visible window chrome.

The uncommitted diff in `BobsDeskView.swift` does all of the following:

**Chrome dissolution.** `WindowTransparencyConfigurator.applyChromelessConfiguration(to:avatarOnly:)` now **removes `.titled` from `styleMask`** in avatar-only mode (in addition to `.fullSizeContentView`). Dropping `.titled` dissolves `NSThemeFrame` entirely — which is the only reliable way on macOS 14 to kill the system-material backdrop that `.containerBackground(.clear, for: .window)` would handle on macOS 15+. `stripBackdrop(under:)` also *removes* any `NSVisualEffectView` it finds rather than hiding it, and sets `layer.backgroundColor = nil` on every walked view. `hasShadow` is conditional: true in full mode, false in avatar-only. Full mode still inserts `.titled` so its chrome remains correct.

**Translucent bubble styling.** New constants `bubbleFill = Color.white.opacity(0.48)` and `bubbleStroke = Color.black.opacity(0.18)`; both multiplied by the caller's `surfaceOpacity`. Soft `.shadow(radius: 10)`. Drops the old hard-stroke comic look.

**Dynamic bubble cap via `GeometryReader`.** `avatarOnlyLayout` is now wrapped in a `GeometryReader`; the body computes:

```swift
let reserved = 160 /*Bob*/ + 56 /*input*/ + 10+10+10+12 /*gaps*/ = 258
let bubbleCap = max(minBubbleHeight, proxy.size.height - reserved)
```

…and passes that into `speechBubbleView(maxHeight:)`. `.layoutPriority(2)` on `draggablePortrait` and `compactInputBubble` guarantees Bob and the input reserve their intrinsic size first — the bubble gives up space, not them. Explicit `Spacer().frame(height: N)` between each row (replaces the old `VStack(spacing: 4)` + `Spacer(minLength: 8)` + trailing `.padding(.bottom, 12)` mix).

**Always-scrolling bubble.** `speechBubbleView` dropped the `.infinity` sentinel + `.fixedSize(vertical: true)` branch. It now **always** wraps text in a `ScrollView(.vertical, showsIndicators: false)` capped at `maxHeight`. The bubble frame is stable as tokens stream in; long replies scroll inside. Removed the `value: rawText` animation that was thrashing per-token.

**Generalized `ComicBubbleShape` — one shape, two tail directions.** Added `enum BubbleTailDirection { case up, down }`. The shape now switches `bodyRect` origin/height and tail `baseY`/`tipY` based on direction. Default `tailDirection = .down` so all existing call sites still work. `tailHeight` default tightened 18 → 14.

**Input pill gains an upward tail.** `compactInputBubble` swapped its `Capsule` background/overlay for `ComicBubbleShape(tailAnchorX: 0.28, cornerRadius: 20, tailWidth: 12, tailHeight: 10, tailDirection: .up)`. `.padding(.top, 9 + 10)` reserves the 10pt the tail needs so the TextField never bleeds into the tail region.

**Window `avatarMinSize` raised 320 → 340.** Matches the view's `.frame(minHeight: 340)` so the layout engine never has to compress rows in a borderless window (gpt-5.4 peer caught this mismatch).

## 8. Non-negotiable architecture rules (repeat from CLAUDE.md)

If you break one of these, the user will revert the change. Do not argue with the rule — if you want to break it, open a discussion first.

1. **Endpoint is `/api/chat`** (Ollama native), never `/v1/chat/completions`.
2. **`stream: false`** everywhere. Gemma 4 + streaming + tool calls is broken.
3. **Flat tool schemas.** Single-level properties. No nested objects or arrays anywhere in the tool spec.
4. **No auto-approve.** Modal NSAlert for every write. Path policy for sensitive dirs.
5. **No Python subprocess, no MCP, no Electron, no LangChain, no Hermes Agent.** The Swift agent loop owns everything.
6. **`Codable` for tool-call arguments must accept both object and string** — Ollama flips formats between turns (see `JSONValue.swift` and `docs/OLLAMABOB_V1.1_PLAN.md`).
7. **App Sandbox OFF** — required for `Process()` shell execution. Hardened runtime ON with the usage-description + automation-events exceptions declared in `build.sh`'s Info.plist heredoc.
8. **`LSUIElement` true** — menu-bar only, no Dock icon.
9. **Persona ≠ operating rules.** Safety rules live in `BobOperatingRules` and must always be re-prepended. Voice/tone is editable; safety is not.

## 9. Known wire-format + runtime gotchas

- **Multi-turn `arguments` bug** — second tool call in a conversation may return `arguments` as a JSON string even though the first was an object. `JSONValue` must handle both.
- **Gemma 4 special chars** — backticks, braces, regex inside tool argument strings can crash the parser. Keep tool schemas simple.
- **Ollama process memory** — use `ps -axo rss=,command=`; `comm=` truncates at 16 chars.
- **AppleScript -1743 after user approves** — caused by missing `NSAppleEventsUsageDescription`. Fixed this session. If it resurfaces on rebuild, suspect ad-hoc-signing churn (§7B).
- **rembg for alpha cutouts** — `uv tool run --from 'rembg[cpu,cli]' rembg i IN OUT`. Do NOT try to use a generative image model — it paints a background instead of producing true alpha.
- **Chromeless SwiftUI windows need three pieces together** — `WindowDragHandle` NSView for dragging, `.resizable` styleMask + `hasShadow=true`, `.windowResizability(.contentMinSize)` with min + ideal frame. Remove any one and the window breaks.
